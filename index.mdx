fast_float is a header-only C++ library providing high-performance implementations of the C++17 `std::from_chars` functions for `float`, `double`, fixed-width floating-point types, and all standard integer types. It delivers exact IEEE 754 rounding, zero memory allocations, and no exceptions — parsing random floating-point numbers at up to 1 GB/s.

Used in production by GCC 12, Chromium, WebKit/Safari, Redis, MySQL, DuckDB, Apache Arrow, and ClickHouse.

## Why fast_float?

fast_float provides the fastest correct floating-point parser available for C++. It is a drop-in replacement for `std::from_chars` that works on C++11 and later, with optional `constexpr` support in C++20.

## How it works

<Steps>
  <Step title="Include the header">
    Copy the single header or add the repository as a CMake dependency. No build step required.
  </Step>
  <Step title="Call from_chars">
    Pass a character range and a reference to your target variable. The function returns a result struct with a pointer and an error code.
  </Step>
  <Step title="Check the result">
    Verify `answer.ec == std::errc()` (or use the C++26-style `bool` conversion) and read the parsed value.
  </Step>
</Steps>

```cpp fast_float_example.cpp
#include "fast_float/fast_float.h"
#include <iostream>
#include <string>

int main() {
  std::string input = "3.1416 xyz";
  double result;
  auto answer = fast_float::from_chars(input.data(), input.data() + input.size(), result);
  if (answer.ec != std::errc()) {
    std::cerr << "parsing failure\n";
    return EXIT_FAILURE;
  }
  std::cout << "parsed: " << result << "\n"; // parsed: 3.1416
  return EXIT_SUCCESS;
}
```

<Note>
  fast_float requires C++11 or later. Constexpr evaluation is available in C++20. Fixed-width floating-point types (`std::float16_t`, `std::bfloat16_t`, etc.) require C++23.
</Note>
