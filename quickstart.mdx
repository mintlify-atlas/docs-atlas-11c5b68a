---

This guide walks you through parsing a floating-point number from a string using fast_float. The entire API surface you need is a single function, `fast_float::from_chars`, and a two-field result struct.

<Steps>
  <Step title="Include the header">
    fast_float is header-only. Add its `include/` directory to your include path (see [Installation](/installation)), then include the main header:

    ```cpp
    #include "fast_float/fast_float.h"
    ```

    No other files need to be compiled or linked.
  </Step>

  <Step title="Call fast_float::from_chars">
    Pass a pointer to the start of the character sequence, a pointer one past the end, and a reference to the variable that receives the parsed value:

    ```cpp
    #include "fast_float/fast_float.h"
    #include <iostream>
    #include <string>

    int main() {
      std::string input = "3.1416 xyz ";
      double result;
      auto answer = fast_float::from_chars(input.data(), input.data() + input.size(), result);
      if (answer.ec != std::errc()) { std::cerr << "parsing failure\n"; return EXIT_FAILURE; }
      std::cout << "parsed the number " << result << std::endl;
      return EXIT_SUCCESS;
    }
    ```

    `from_chars` reads as many characters as it can to form a valid number and stops. Characters after the number (like the space and `xyz` above) are ignored by the parser and left for you to handle.
  </Step>

  <Step title="Check the result struct">
    `from_chars` returns a `fast_float::from_chars_result` with two fields:

    | Field | Type | Meaning |
    |-------|------|---------|
    | `ptr` | `char const *` | Points to the first character that was **not** consumed by the parse. |
    | `ec`  | `std::errc` | `std::errc()` (zero) on success; `std::errc::result_out_of_range` for overflow/underflow; `std::errc::invalid_argument` if no number was found at all. |

    A successful parse satisfies `answer.ec == std::errc()`. After a successful parse, `answer.ptr` points just past the number, making it easy to continue scanning the string.
  </Step>
</Steps>

## Alternative: C++26-style bool conversion

Prior to C++26, checking `ec` against `std::errc()` is the standard idiom. As an extension, fast_float also supports the C++26 convention of converting the result directly to `bool`:

```cpp
#include "fast_float/fast_float.h"
#include <iostream>
#include <string>

int main() {
  std::string input = "3.1416 xyz ";
  double result;
  if (auto answer = fast_float::from_chars(input.data(), input.data() + input.size(), result)) {
    std::cout << "parsed the number " << result << std::endl;
    return EXIT_SUCCESS;
  }
  std::cerr << "failed to parse " << input << std::endl;
  return EXIT_FAILURE;
}
```

This works even when compiling in C++11 mode — the `explicit operator bool()` is provided by the library itself.

## Parsing delimited numbers

Because `answer.ptr` points past the last consumed character, you can chain calls to parse a series of comma-separated (or otherwise delimited) numbers without copying or splitting the string:

```cpp
std::string input = "234532.3426362,7869234.9823,324562.645";
double result;
auto answer = fast_float::from_chars(input.data(), input.data() + input.size(), result);
if (answer.ec != std::errc()) {
  // check error
}
// result == 234532.3426362
if (answer.ptr[0] != ',') {
  // unexpected delimiter
}
answer = fast_float::from_chars(answer.ptr + 1, input.data() + input.size(), result);
if (answer.ec != std::errc()) {
  // check error
}
// result == 7869234.9823
if (answer.ptr[0] != ',') {
  // unexpected delimiter
}
answer = fast_float::from_chars(answer.ptr + 1, input.data() + input.size(), result);
if (answer.ec != std::errc()) {
  // check error
}
// result == 324562.645
```

This pattern is efficient for CSV parsing and similar tasks because no intermediate strings are allocated.
